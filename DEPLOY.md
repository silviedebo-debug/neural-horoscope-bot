# Деплой на Railway.app — пошаговая инструкция для новичка

Эта инструкция рассчитана на то, что вы никогда раньше не разворачивали проекты
на облачном хостинге. Ничего не пропускайте, даже если кажется очевидным.

---

## Шаг 0. Что вам понадобится заранее

- [ ] Аккаунт на GitHub (github.com) — если нет, зарегистрируйтесь, это бесплатно.
- [ ] Установленный Git на компьютере — проверить: откройте терминал (Windows: PowerShell,
      Mac/Linux: Terminal) и введите `git --version`. Если ошибка — скачайте с git-scm.com.
- [ ] Токен бота от @BotFather (если ещё нет — см. Шаг 1).
- [ ] Аккаунт на console.apify.com и API-токен (см. Шаг 2).
- [ ] Ваш личный Telegram chat_id (см. Шаг 3).

---

## Шаг 1. Получить токен Telegram-бота

1. Откройте Telegram, найдите бота **@BotFather**.
2. Отправьте ему команду `/newbot` (если бот уже создан ранее — `/mybots` → выберите
   бота → `API Token`).
3. Следуйте инструкциям: укажите имя бота (например, "Нейрогороскоп") и username
   (должен заканчиваться на `bot`, например `neurohoroscope_test_bot`).
4. BotFather пришлёт токен вида `8275273528:AAGvLb6QhxuYymt82QNy8...`. Скопируйте
   его — он понадобится на Шаге 6. **Никому его не показывайте и не публикуйте.**

## Шаг 2. Получить токен Apify

1. Зарегистрируйтесь на console.apify.com (есть бесплатный тариф).
2. В левом меню: **Settings** → **API & Integrations** → **API tokens**.
3. Скопируйте **Personal API token** (или создайте новый).
4. Убедитесь, что актор `alex_claw/telegram-channel-scraper` вам доступен
   (можно найти через поиск в Apify Store и один раз запустить вручную,
   чтобы убедиться, что он у вас работает).

## Шаг 3. Узнать свой Telegram chat_id

1. В Telegram найдите бота **@userinfobot**.
2. Отправьте ему `/start`.
3. Он пришлёт сообщение с полем `Id: 123456789` — это ваш `ADMIN_CHAT_ID`.

## Шаг 4. Скачать файлы проекта и подготовить папку

1. Скачайте все файлы проекта (архив, приложенный к этому сообщению) и распакуйте
   в отдельную папку на компьютере, например `horoscope-bot`.
2. Убедитесь, что в папке есть файлы: `main.py`, `bot/` (папка), `tests/` (папка),
   `requirements.txt`, `requirements-dev.txt`, `Procfile`, `.gitignore`, `.env.example`,
   `README.md`, `SECURITY.md`.
3. **Не создавайте `.env` файл с реальными токенами внутри этой папки перед тем,
   как загрузить её в Git** — сначала убедитесь, что `.gitignore` на месте
   (он не даст `.env` попасть в репозиторий, но лучше перестраховаться и создать
   `.env` только для локального теста, отдельно от этапа загрузки в GitHub).

## Шаг 5. Загрузить проект на GitHub

Откройте терминал внутри папки проекта (`cd путь/к/horoscope-bot`) и выполните
по очереди:

```bash
git init
git add .
git status
```

**Важно:** после `git status` посмотрите список файлов — там **не должно быть**
`.env`, `*.db`, `*.log`. Если они там есть — значит `.gitignore` не сработал,
остановитесь и разберитесь, прежде чем продолжать (не коммитьте секреты).

Если всё чисто:

```bash
git commit -m "Initial commit: neurohoroscope bot"
```

Дальше:
1. Зайдите на github.com → нажмите **New repository** (зелёная кнопка справа сверху).
2. Назовите репозиторий, например `neurohoroscope-bot`. Оставьте **Private**
   (личный пет-проект — не обязательно делать публичным сразу; когда будете
   готовы показать работодателю, можно переключить на Public в Settings репозитория).
3. **Не** ставьте галочку "Add README" — у вас он уже есть.
4. Нажмите **Create repository**.
5. GitHub покажет команды вида:

```bash
git remote add origin https://github.com/ваш-логин/neurohoroscope-bot.git
git branch -M main
git push -u origin main
```

Скопируйте и выполните их в терминале (замените ссылку на ту, что показал GitHub
именно вам). Если попросит логин/пароль — используйте Personal Access Token
вместо пароля (GitHub Settings → Developer settings → Personal access tokens),
либо настройте вход через GitHub Desktop, если работа с токенами непривычна.

## Шаг 6. Создать проект на Railway

1. Зайдите на railway.app, нажмите **Login** → войдите через GitHub-аккаунт
   (это упростит следующий шаг).
2. Нажмите **New Project**.
3. Выберите **Deploy from GitHub repo**.
4. Если Railway попросит доступ к вашим репозиториям — разрешите (можно
   ограничить доступ только нужным репозиторием в настройках GitHub App).
5. Выберите репозиторий `neurohoroscope-bot`.
6. Railway начнёт анализировать проект и предложит задеплоить его как сервис —
   согласитесь (Deploy Now можно нажать позже, сначала настроим переменные).

## Шаг 7. Задать переменные окружения

1. В интерфейсе Railway откройте созданный сервис (кликните на его карточку).
2. Перейдите во вкладку **Variables**.
3. Нажмите **New Variable** и добавьте по одной:
   - `TELEGRAM_TOKEN` = токен из Шага 1
   - `APIFY_API_TOKEN` = токен из Шага 2
   - `ADMIN_CHAT_ID` = ваш id из Шага 3
4. Нажмите **Deploy** (или Railway передеплоит автоматически при сохранении переменных).

**Почему это безопасно:** переменные хранятся в зашифрованном виде на стороне
Railway и никогда не появляются в коде репозитория — именно поэтому `config.py`
читает их через `os.environ`, а не через файл.

## Шаг 8. Убедиться, что сервис — Worker, а не Web Service

Этот бот не принимает входящие HTTP-запросы (использует polling), поэтому ему
**не нужен публичный домен и открытый порт**.

1. Во вкладке **Settings** сервиса найдите раздел **Deploy** → убедитесь, что
   Start Command соответствует строке из `Procfile`: `python main.py`
   (Railway обычно подхватывает `Procfile` автоматически — если нет, впишите
   команду вручную в поле Custom Start Command).
2. Ничего не настраивайте в разделе **Networking** — этому сервису не нужен
   домен, так как он сам ходит в Telegram API, а не принимает запросы извне.

## Шаг 9. Проверить логи

1. Вкладка **Deployments** → откройте последний деплой → **View Logs**.
2. Ищите строки:
   ```
   Конфигурация загружена:
   ...
   База данных инициализирована
   ...
   Бот запущен. Ожидание сообщений...
   ```
3. Если видите `TELEGRAM_TOKEN не задан` или похожее — вернитесь на Шаг 7 и
   проверьте, что переменная сохранена без опечаток в имени.

## Шаг 10. Проверить бота вживую

1. Откройте вашего бота в Telegram, отправьте `/start`.
2. Должно прийти приветствие. В логах Railway должна появиться строка
   `СТАРТ от user_id=...`.
3. Отправьте название знака зодиака (например, "Овен") — должен прийти ответ
   в формате `♈️Овен: ...`.
4. Проверить ежедневную сводку сразу, не дожидаясь 09:00 UTC: временно
   поменяйте `DAILY_REPORT_TIME` в `bot/config.py` на ближайшее время (в UTC!),
   закоммитьте и запушьте изменение — Railway передеплоит автоматически.
   После проверки верните исходное время.

## Шаг 11. Как обновлять бота дальше

Любое изменение кода — стандартный git-процесс:

```bash
git add .
git commit -m "Описание изменения"
git push
```

Railway сам подхватит пуш в `main` и передеплоит сервис — никаких
дополнительных действий на самой платформе не требуется.

## Что делать, если бот "упал" (Railway перезапустил контейнер)

- Проверьте вкладку **Deployments** → там будет видно, было ли аварийное
  завершение процесса, и логи перед падением.
  Ошибка ImportError -> опечатка при переносе структуры папок,
  Ошибка "Unauthorized" от Telegram -> неверный/отозванный токен,
  Ошибка от Apify -> см. логи `apify_service.py`, там подробно логируется каждая попытка.
- SQLite-файл (`bot_data.db`) при пересборке контейнера может обнулиться —
  это ожидаемо на бесплатном тарифе без подключённого постоянного volume.
  Если это критично — добавьте в Railway **Volume** (Settings → Volumes) и
  укажите путь монтирования, совпадающий с `DB_PATH` из переменных окружения.
